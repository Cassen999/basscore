## UI Components

### `Fretboard`
**File:** `src/components/Fretboard.tsx`
**Props:** `iFretboardProps` (optional `coords: iCoords[]`)
**Export:** `export default Fretboard`
**Purpose:** Renders an SVG bass guitar fretboard. Reads fretboard dimensions and string/fret count from `ControlsContext` via `useControls()`. Accepts an array of coordinate objects to render colored dot markers on the fretboard.
**Usage notes:**
- Does not manage its own size — dimensions come from `fretboardConfig` in context
- Pages that use it call `setFretboardConfig` on mount to configure fret/string count for that view
- `coords` are generated by helpers in `src/helpers/fretpoints.tsx`

### `ControlPanel`
**File:** `src/components/ControlPanel.tsx`
**Props:** `iControlProps` — `elements: ReactNode[] | iControlElementGroups[]`, optional `cardProps`
**Export:** `export default ControlPanel`
**Purpose:** A PrimeReact `Card` wrapper that renders grouped or flat lists of control elements (dropdowns, sliders, color pickers, etc.).
**Usage notes:**
- Pass flat `ReactNode[]` for ungrouped controls
- Pass `iControlElementGroups[]` (each with `title` and `elements`) for labeled groups

### `Metronome`
**File:** `src/components/Metronome.tsx`
**Props:** `iMetronome` — `bpm`, `subdivision: tSubdivision`, `isPlaying`, `volume`, `bpMeasure`
**Export:** `export const Metronome`
**Purpose:** Handles all Web Audio API logic (oscillator, gain, interval timer) and renders the beat dot visual. The beat dots reflect the active beat in sync with the audio click.
**Usage notes:**
- Beat count is tracked with both a `useState` (for re-renders) and a `useRef` (for reading inside `setInterval` without stale closure)
- Downbeat (first beat of each measure) plays at 500Hz; other beats at 400Hz
- Resetting `subdivision` or `bpMeasure` resets the beat position to 0
- Stopping resets beat position so the next Start always begins at dot 0

## Contexts

### `ControlsContext`
**File:** `src/contexts/ControlsContext.tsx`
**Hook:** `useControls()` — throws if used outside `ControlsProvider`
**Provider:** Wraps all routes via `HomeContainer`
**Output type:** `iControlsContext`
**Memoized:** Yes — context value is wrapped in `useMemo`

Provides:

| Value | Type | Default | Purpose |
|---|---|---|---|
| `interval` | `tInterval` | `2` | Currently selected interval (2–8) |
| `setInterval` | setter | — | |
| `intervalColors` | `iIntervalColors` | CSS vars | Root, interval, unison note colors with setters |
| `showUnison` | `boolean` | `true` | Toggle unison octave dots |
| `setShowUnison` | setter | — | |
| `displayedScales` | `tScaleType` | `'major'` | Currently selected scale mode |
| `setDisplayedScales` | setter | — | |
| `scaleNoteColor` | `tColorType` | CSS var | Note color for scale view |
| `setScaleNoteColor` | setter | — | |
| `fretboardConfig` | `iFretboardConfig` | see below | Fretboard dimensions and layout |
| `setFretboardConfig` | setter | — | Pages call this on mount to configure fret/string count |

Default `fretboardConfig`: `{ width: 700, height: 200, numFrets: 5, numStrings: 4, fretpointRadius: 8 }`

## Helpers

### `fretboardHelpers`
**File:** `src/helpers/fretboardHelpers.tsx`
**Purpose:** SVG positioning math — `getX`, `getY`, `mapFretPoints`. Used by `Fretboard` to convert coordinate data into SVG circle elements.

### `fretpoints`
**File:** `src/helpers/fretpoints.tsx`
**Purpose:** Generates `iCoords[]` arrays for a given scale or interval. Used by `Scales` and `Intervals` pages to produce the dot data passed to `Fretboard`.
